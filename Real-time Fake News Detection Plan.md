## 1. CONSTRAINTS
<!-- Read this section fully before writing a single line of code. -->

### 1.1 SpacetimeDB Hard Rules

| Rule ID | Constraint | Consequence of violation |
|---|---|---|
| SDB-01 | Reducers MUST be fully deterministic | State corruption on replay |
| SDB-02 | Reducers MUST NOT call `Math.random()`, `Date.now()`, or any HTTP | Non-determinism breaks ACID |
| SDB-03 | Use `ctx.timestamp` (not `Date.now()`) for all timestamps inside reducers | Divergent state across nodes |
| SDB-04 | Procedures MUST NOT mutate tables directly — use `ctx.withTx()` | Runtime error |
| SDB-05 | HTTP calls inside procedures MUST occur OUTSIDE any open `withTx` block | Database lock contention |
| SDB-06 | Authentication MUST use `ctx.sender` only — never trust an identity passed as a function argument | Authorization bypass |
| SDB-07 | Global/module-level mutable variables in server modules are FORBIDDEN | Undefined behavior on re-execution |
| SDB-08 | The schema export `fakeNewsSchema` is the ONLY schema; do not create additional schemas | Schema fragmentation |

### 1.2 Project Scope Rules

| Rule ID | Constraint |
|---|---|
| SCOPE-01 | The only tables permitted are: `article_metadata`, `detection_inference`, `system_user` |
| SCOPE-02 | Do NOT add an `ArchivalQueue` table or any `scheduleAt` column |
| SCOPE-03 | Do NOT add AWS SDK, S3, Firehose, Glue, Snowflake, or Athena dependencies |
| SCOPE-04 | NLP classification MUST use the Claude API prompt specified in TASK-03 unless overridden by a human |
| SCOPE-05 | The React dashboard MUST use `useTable()` subscriptions — no `fetch()`, `axios`, or REST polling |

### 1.3 Classification Enum
The only valid values for `DetectionInference.classification` are:

```
"Real" | "Fake" | "Unverified"
```

Any NLP backend response that does not map to one of these three values MUST default to `"Unverified"`.

### 1.4 Valid Role Enum
The only valid values for `SystemUser.role` are:

```
"Admin" | "Analyst" | "API_Agent"
```

### 4.1 — Schema (`spacetimedb/src/index.ts`)

```typescript
import { schema, table, t, SenderError, TimeDuration } from 'spacetimedb/server';

const ArticleMetadata = table(
  { name: 'article_metadata', public: true },
  {
    id:           t.u64().primaryKey().autoInc(),
    source_url:   t.string().unique(),
    domain:       t.string().index('btree'),
    headline:     t.string(),
    published_at: t.timestamp(),
    ingested_at:  t.timestamp(),
  }
);

const DetectionInference = table(
  { name: 'detection_inference', public: true },
  {
    article_id:           t.u64().unique(),
    is_processed:         t.bool(),
    classification:       t.string(),  // enum: "Real" | "Fake" | "Unverified"
    confidence_score:     t.f32(),
    model_version:        t.string(),
    flagged_by_newsguard: t.bool(),
  }
);

const SystemUser = table(
  { name: 'system_user', public: true },
  {
    identity:   t.identity().primaryKey(),
    role:       t.string(),  // enum: "Admin" | "Analyst" | "API_Agent"
    created_at: t.timestamp(),
  }
);

export const fakeNewsSchema = schema({
  ArticleMetadata,
  DetectionInference,
  SystemUser,
});
```

### 4.2 — Ingestion Reducer

```typescript
export const ingest_news_article = fakeNewsSchema.reducer(
  {
    source_url:     t.string(),
    domain:         t.string(),
    headline:       t.string(),
    published_time: t.timestamp(),
  },
  (ctx, { source_url, domain, headline, published_time }) => {

    // AUTH: ctx.sender is cryptographically verified — never pass identity as an arg (SDB-06)
    const user = ctx.db.SystemUser.identity.find(ctx.sender);
    if (!user || user.role !== 'API_Agent') {
      throw new SenderError("Unauthorized: caller is not an API_Agent.");
    }

    // IDEMPOTENCY: silently drop duplicate URLs — safe for pipeline replay
    if (ctx.db.ArticleMetadata.source_url.find(source_url)) return;

    const article = ctx.db.ArticleMetadata.insert({
      id:           0n,  // autoInc handles the actual value
      source_url,
      domain,
      headline,
      published_at: published_time,
      ingested_at:  ctx.timestamp,  // deterministic (SDB-03)
    });

    ctx.db.DetectionInference.insert({
      article_id:           article.id,
      is_processed:         false,
      classification:       "Unverified",
      confidence_score:     0.0,
      model_version:        "pending",
      flagged_by_newsguard: false,
    });
  }
);
```

### 4.3 — Finalize Inference Reducer

```typescript
// Called exclusively by trigger_nlp_analysis after the HTTP call completes.
// Marking it export allows the CLI to call it directly for testing (e.g., TASK-06 VERIFY).
export const finalize_inference = fakeNewsSchema.reducer(
  {
    article_id:     t.u64(),
    classification: t.string(),
    confidence:     t.f32(),
    model_version:  t.string(),
  },
  (ctx, { article_id, classification, confidence, model_version }) => {
    const record = ctx.db.DetectionInference.article_id.find(article_id);
    if (!record) throw new SenderError(`No inference record for article_id ${article_id}.`);

    // Enforce classification enum — any unknown value becomes "Unverified"
    const safeClass = ["Real", "Fake", "Unverified"].includes(classification)
      ? classification
      : "Unverified";

    ctx.db.DetectionInference.article_id.update({
      ...record,
      is_processed:    true,
      classification:  safeClass,
      confidence_score: confidence,
      model_version,
    });
  }
);
```

### 4.4 — NLP Analysis Procedure

```typescript
// CANONICAL NLP PROMPT — do not alter wording without human approval
const NLP_SYSTEM_PROMPT = `You are a fact-checking classifier for a real-time disinformation detection system.
Analyze the provided news headline for credibility signals.
Respond ONLY with a single valid JSON object — no preamble, no markdown, no explanation:
{"predicted_class":"Real"|"Fake"|"Unverified","confidence_score":0.0-1.0,"model_version":"claude-sonnet-4-20250514"}`;

export const trigger_nlp_analysis = fakeNewsSchema.procedure(
  { article_id: t.u64(), text_payload: t.string() },
  t.unit(),
  (ctx, { article_id, text_payload }) => {
    try {
      // HTTP MUST be outside withTx — rule SDB-05
      const response = ctx.http.fetch(process.env.NLP_ENDPOINT!, {
        method: "POST",
        headers: {
          "Content-Type":  "application/json",
          "x-api-key":     process.env.ANTHROPIC_API_KEY!,
          "anthropic-version": "2023-06-01",
        },
        body: JSON.stringify({
          model:      "claude-sonnet-4-20250514",
          max_tokens: 128,
          system:     NLP_SYSTEM_PROMPT,
          messages: [{ role: "user", content: `Classify: "${text_payload}"` }],
        }),
        timeout: TimeDuration.fromMillis(8000),
      });

      if (response.status !== 200) {
        console.error(`NLP endpoint returned ${response.status} for article_id ${article_id}`);
        return {};
      }

      const raw = response.json();
      // Claude wraps the response — extract the text block
      const textBlock = raw?.content?.find((b: any) => b.type === "text")?.text ?? "";

      let parsed: any;
      try {
        parsed = JSON.parse(textBlock);
      } catch {
        console.error(`Failed to parse NLP JSON for article_id ${article_id}: ${textBlock}`);
        return {};
      }

      // Write result — mutation inside withTx (SDB-04)
      ctx.withTx((txCtx) => {
        finalize_inference(txCtx, {
          article_id,
          classification: parsed.predicted_class ?? "Unverified",
          confidence:     parsed.confidence_score ?? 0.0,
          model_version:  parsed.model_version   ?? "unknown",
        });
      });

    } catch (e) {
      console.error(`trigger_nlp_analysis failed for article_id ${article_id}:`, e);
    }

    return {};
  }
);
```

### 4.5 — Analyst Override Reducer

```typescript
// Allows "Admin" or "Analyst" roles to manually correct a classification.
export const override_classification = fakeNewsSchema.reducer(
  {
    article_id:     t.u64(),
    classification: t.string(),
    rationale:      t.string(),
  },
  (ctx, { article_id, classification, rationale }) => {
    const user = ctx.db.SystemUser.identity.find(ctx.sender);
    if (!user || !["Admin", "Analyst"].includes(user.role)) {
      throw new SenderError("Unauthorized: override requires Admin or Analyst role.");
    }

    const safeClass = ["Real", "Fake", "Unverified"].includes(classification)
      ? classification
      : "Unverified";

    const record = ctx.db.DetectionInference.article_id.find(article_id);
    if (!record) throw new SenderError(`No inference record for article_id ${article_id}.`);

    ctx.db.DetectionInference.article_id.update({
      ...record,
      classification:  safeClass,
      model_version:   `manual-override:${user.identity}`,
      is_processed:    true,
    });

    // rationale is logged to stdout — visible in spacetime logs
    console.log(`[OVERRIDE] article_id=${article_id} → ${safeClass} | reason: ${rationale}`);
  }
);
```

### 4.6 — React Dashboard (`src/dashboard/LiveThreatDashboard.tsx`)

```typescript
import { useTable } from '../module_bindings';
import { tables }   from '../module_bindings';
import { useState } from 'react';

type FilterMode = 'All' | 'Fake' | 'Unverified';

export function LiveThreatDashboard() {
  const [filter, setFilter] = useState<FilterMode>('Fake');

  // Local in-memory cache — auto-synced by SpacetimeDB WebSocket (SCOPE-05)
  const allInferences = useTable(tables.DetectionInference);
  const allArticles   = useTable(tables.ArticleMetadata);

  const visibleRows = allInferences.filter(row => {
    if (filter === 'All') return true;
    return row.classification === filter;
  });

  const metaMap = new Map(allArticles.map(a => [a.id.toString(), a]));

  const counts = {
    fake:       allInferences.filter(r => r.classification === 'Fake').length,
    real:       allInferences.filter(r => r.classification === 'Real').length,
    unverified: allInferences.filter(r => r.classification === 'Unverified').length,
  };

  return (
    <div className="dashboard">
      <header>
        <h1>Disinformation Detection — Live Feed</h1>
        <div className="stats">
          <span className="stat fake">Fake: {counts.fake}</span>
          <span className="stat real">Real: {counts.real}</span>
          <span className="stat unverified">Unverified: {counts.unverified}</span>
        </div>
      </header>

      <nav className="filters">
        {(['All', 'Fake', 'Unverified'] as FilterMode[]).map(mode => (
          <button
            key={mode}
            className={filter === mode ? 'active' : ''}
            onClick={() => setFilter(mode)}
          >
            {mode}
          </button>
        ))}
      </nav>

      <div className="threat-grid">
        {visibleRows.map(row => {
          const meta = metaMap.get(row.article_id.toString());
          return (
            <div key={Number(row.article_id)} className={`threat-card ${row.classification.toLowerCase()}`}>
              <h4>{meta?.headline ?? '—'}</h4>
              <p className="domain">{meta?.domain}</p>
              <div className="confidence">
                Confidence: <strong>{(row.confidence_score * 100).toFixed(1)}%</strong>
              </div>
              <div className="badges">
                <span className={`badge ${row.classification.toLowerCase()}`}>
                  {row.classification}
                </span>
                {row.flagged_by_newsguard && (
                  <span className="badge newsguard">NewsGuard ✓</span>
                )}
              </div>
            </div>
          );
        })}
      </div>
    </div>
  );
}
```

## 5. ERROR HANDLING REFERENCE

| Situation | Required Action |
|---|---|
| `spacetime publish` fails with schema error | Fix the schema in §4.1, do not add new tables to work around errors |
| Claude API returns non-JSON text | Log the raw response, default `classification` to `"Unverified"`, continue |
| `ingest.ts` article is missing `url`, `source`, or `title` | Log `[SKIP] missing field: <field>` and continue — do not throw |
| `finalize_inference` called for unknown `article_id` | Reducer throws `SenderError` — this is correct behavior, log and investigate |
| Dashboard shows stale data | Confirm WebSocket is connected; do NOT add a polling interval as a fix |
| NLP endpoint timeout (>8s) | Procedure logs the error and returns `{}` — article stays `"Unverified"` |
| Classification value outside enum | `finalize_inference` silently coerces to `"Unverified"` (see §4.3) |
