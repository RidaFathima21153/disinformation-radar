# AGENT PROMPT — Enhanced Frontend for `fake-news-engine`

## 0. YOUR MISSION

You are building a **visually stunning, information-rich React dashboard** for a real-time disinformation detection system called `fake-news-engine`. The backend is powered by **SpacetimeDB** (in-memory, WebSocket-first) and classifies news articles via the Claude API as `"Real"`, `"Fake"`, or `"Unverified"`.

A minimal dashboard already exists. Your job is to **completely replace and upgrade** it with a premium, editorial-style public-awareness dashboard that includes rich data visualizations, animated charts, and actionable analyst tools — all streaming live from SpacetimeDB without any REST polling.

## 2. DATA SCHEMA (from `src/module_bindings/types.ts`)

These are the **exact TypeScript types** generated by SpacetimeDB. Field names are camelCase.

```typescript
// Article metadata — one row per ingested news article
type ArticleMetadata = {
  id: bigint;           // unique article ID (u64)
  sourceUrl: string;    // original article URL
  domain: string;       // e.g. "bbc.com", "infowars.com"
  headline: string;     // article headline text
  publishedAt: bigint;  // timestamp (microseconds since epoch)
  ingestedAt: bigint;   // timestamp (microseconds since epoch)
};

// AI classification result — one row per article (linked by articleId)
type DetectionInference = {
  articleId: bigint;         // foreign key → ArticleMetadata.id
  isProcessed: boolean;      // false = pending NLP, true = classified
  classification: string;    // "Real" | "Fake" | "Unverified"
  confidenceScore: number;   // 0.0 – 1.0 float
  modelVersion: string;      // e.g. "claude-sonnet-4-20250514" or "pending"
  flaggedByNewsguard: boolean;
};

// System users (Admin / Analyst / API_Agent)
type SystemUser = {
  identity: string;   // SpacetimeDB identity hex
  role: string;       // "Admin" | "Analyst" | "API_Agent"
  createdAt: bigint;
};
```

## 3. HOW TO READ DATA (CRITICAL — READ CAREFULLY)

All data comes from `useTable()`. **Do not use `fetch()`, `axios`, `useEffect` with intervals, or any REST endpoint.**

```typescript
import { tables }   from '../module_bindings';
import { useTable } from 'spacetimedb/react';

// Inside a component:
const [allInferences] = useTable(tables.DetectionInference);
const [allArticles]   = useTable(tables.ArticleMetadata);

// These arrays update in real time via WebSocket — treat them like useState values.
```

### Joining data
```typescript
// Build a lookup map from articleId → metadata
const metaMap = new Map(allArticles.map(a => [a.id.toString(), a]));

// Then for any inference row:
const meta = metaMap.get(row.articleId.toString());
```

### `useDerivedStats.ts` pattern
Create a custom hook that accepts `allInferences` and `allArticles` arrays and returns all derived metrics, so components stay clean:

```typescript
export function useDerivedStats(
  inferences: DetectionInference[],
  articles: ArticleMetadata[]
) {
  return useMemo(() => {
    const processed  = inferences.filter(r => r.isProcessed);
    const fake       = inferences.filter(r => r.classification === 'Fake');
    const real       = inferences.filter(r => r.classification === 'Real');
    const unverified = inferences.filter(r => r.classification === 'Unverified');
    const pending    = inferences.filter(r => !r.isProcessed);

    const totalProcessed    = processed.length;
    const fakeRate          = totalProcessed > 0 ? (fake.length / totalProcessed) * 100 : 0;
    const avgConfidence     = processed.length > 0
      ? processed.reduce((s, r) => s + r.confidenceScore, 0) / processed.length
      : 0;
    const uniqueDomains     = new Set(articles.map(a => a.domain)).size;

    return { fake, real, unverified, pending, processed, fakeRate, avgConfidence, uniqueDomains };
  }, [inferences, articles]);
}
```

## 9. IMPLEMENTATION RULES

| Rule | Requirement |
|------|------------|
| DATA-01 | All data from `useTable()` only — no `fetch()`, `axios`, or REST |
| DATA-02 | Join `DetectionInference` → `ArticleMetadata` via `articleId → id` |
| DATA-03 | `publishedAt` and `ingestedAt` are in **microseconds** — divide by 1000 for `Date` |
| PERF-01 | Wrap all derived computations in `useMemo()` with appropriate dependencies |
| PERF-02 | Do not re-render chart components on every `useTable` update — memoize chart data |
| UI-01 | All classification values MUST be one of: `"Real"` / `"Fake"` / `"Unverified"` — handle unexpected values gracefully by displaying "Unknown" |

| UI-03 | `overrideClassificationReducer.call()` can throw — wrap in try/catch and show an error toast |
| STYLE-01 | Never use `!important` — use CSS specificity correctly |
| STYLE-02 | All new CSS must be added to `index.css` (not inline styles), except dynamic values (e.g. `width: ${pct}%`) |
